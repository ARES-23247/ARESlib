The ARESLib Standard

Coding rules for championship-grade FTC software. Compliance is mandatory.

1. Zero-Allocation Periodic

Allocating objects inside update() or periodic() is strictly forbidden. This prevents Garbage Collection jitter in high-frequency control loops.

void update() {
  Vector2d pos = new Vector2d(x, y);
  process(pos);
}
private final Vector2d pos = new Vector2d();

void update() {
  pos.setX(x); pos.setY(y);
  process(pos);
}

2. Never Nester

Avoid deep logic nesting. Use Guard Clauses (early returns) to keep the primary execution path at the lowest indentation level possible.

if (isEnabled) {
  if (hasTarget) {
    runMechanism();
  }
}
if (!isEnabled || !hasTarget) return;

runMechanism();

3. Unit Safety

Explicitly document and use standard units (Meters, Radians, Volts). Avoid raw motor "power" percentages in core algorithms.

4. Deterministic Logging

All subsystem inputs must be logged via AresAutoLogger to enable frame-accurate log replay.
