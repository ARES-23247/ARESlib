Elite Developer Suite

Professionalize your FTC workflow with the MARS/ARES VS Code Extension.

Developing championship-grade code requires more than just a text editor. The **MARSLib VS Code Extension** is a unified "Mission Control" center designed to bridge the gap between static code and live robot operations. Whether you're debugging on the field or simulating in the pits, this toolkit ensures your software is as reliable as your hardware.

📥 Download & Installation

The extension is distributed as a .vsix package. Download the latest version from our GitHub Releases. To install, drag the file into VS Code or use the "Install from VSIX..." option in the Extensions panel.

1. The Virtual Dashboard (NT4)

Stop squinting at the Driver Station logs. Our **Virtual Dashboard** provides a high-fidelity, real-time view of your robot's internals using the NetworkTables 4 (NT4) protocol. It's optimized for the ARESLib @AutoLog system, meaning every field marked for logging appears automatically.

  • Battery Health: Visual voltage sag monitoring to prevent brownouts.
  • Live Odometry: Real-time field-coordinate tracking with a custom field layout.
  • Fault Alerts: Instant visual feedback when the AresFaultManager detects a hardware failure.

Hardware Integrity Map

A visual representation of the Control Hub and Expansion Hub wiring, highlighting device connectivity and CAN bus health.

2. Project Doctor: Real-time Audit

The **Project Doctor** is a background linter that enforces the "Zero-Allocation" standards required for 20ms deterministic loops. If you accidentally instantiate an object in a hot loop or use non-thread-safe methods, the Doctor will highlight it in red and offer a quick-fix.

  • Allocation Guard: Detects new keyword usage in periodic() methods.
  • Hot-Loop Linting: Warns about expensive string concatenations or math operations.
  • Command Audits: Ensures all commands have addRequirements() correctly defined.

3. Physics Test Generator

Tired of writing boilerplate for unit tests? The extension features an **Interactive Test Wizard**. Simply right-click a Subsystem file and select "Generate MARS Physics Test" to instantly scaffold a JUnit 5 test environment integrated with the dyn4j simulator.

// Automatically generated by the extension
@Test
void testMechanismSafety() {
    simulate(2.0, () -> { // Runs 2 seconds of physics
        subsystem.setVoltage(12.0);
        assertTrue(subsystem.atSoftLimit());
    });
}

4. Robot Heartbeat Toolbar

Located at the bottom of your IDE, the Heartbeat bar provides persistent status updates on your connection to the REV Control Hub.

  • RIO/Hub Ping: Latency monitoring (Green < 5ms, Yellow > 20ms).
  • Git Sync: Alerts you if your local code is out of sync with the team's remote.
  • Deployment History: One-click navigation to previous successful builds.

Pairing Note

For the telemetry tools to function, ensure your laptop is paired via the Robot Heartbeat status bar at the bottom of VS Code. A broken link icon indicates no active pairing with the Control Hub.
